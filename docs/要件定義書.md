## 1. プロジェクト概要

### 1.1 プロジェクト名
**TRYPAS（トライパス）Web版ゲームアプリケーション**

### 1.2 プロジェクトの背景

トライパスは、思考力・記憶力・創造力を鍛えることができる日本発祥のボードゲームです。1億以上の開始陣形があり、必ず14手以内で勝負が決まる奥深いゲームでありながら、ルールは非常にシンプルで誰でもすぐにプレイできる特徴があります。

現状、物理的なボードゲームとしては存在していますが、デジタル版が存在しないため、以下の課題があります：

- オンラインで気軽にプレイできない
- 相手がいない時に練習できない
- ルールを知らない人への普及が難しい
- 初心者が学習する手段が限られている

### 1.3 プロジェクトの目的

本プロジェクトは、トライパスゲームをWebブラウザ上でプレイ可能にすることで、以下を実現します：

1. **アクセシビリティの向上**  
   インターネット環境があれば、いつでもどこでもプレイ可能

2. **普及促進**  
   ルール説明機能により、初心者でも容易に理解できる

3. **練習環境の提供**  
   CPU対戦やソロモードにより、相手がいなくても練習可能

4. **ゲーム体験の向上**  
   デジタル化により、スコア計算やルール判定が自動化され、ゲームに集中できる
   BGMや効果音による臨場感のある演出

### 1.4 プロジェクトのゴール

- **短期ゴール（リリース時）**
   - ローカル2人対戦が可能
   - CPU対戦（3段階の難易度）が可能
   - ソロモードでの練習が可能
   - ルール説明機能の実装
   - BGM/SEの実装と制御

- **中期ゴール（リリース後3ヶ月）**
   - ユーザーフィードバックに基づく改善
   - パフォーマンス最適化
   - バグ修正と安定化

- **長期ゴール（リリース後6ヶ月〜）**
   - オンライン対戦機能
   - ランキングシステム
   - リプレイ機能の強化（保存・共有）
   - カスタム陣形作成機能

## 4. ユーザー要件

### 4.1 ユーザーストーリー

#### US-001: 初心者がルールを学ぶ
> **As a** 初心者  
> **I want to** ルール説明を読む  
> **So that** ゲームの遊び方を理解できる

**受入基準:**
- タイトル画面およびゲーム中から「ルール説明」ボタンでルールを表示できる
- ルール説明モーダルのタイトルは「TRYPAS SOLOPLAYの遊び方」とする
- コマの色と得点が視覚的に分かる
- 移動ルールが図解付きで説明されている

#### US-002: プレイヤーがローカル対戦する
> **As a** プレイヤー  
> **I want to** 同じデバイスで2人対戦する  
> **So that** 友人と対面でプレイできる

**受入基準:**
- タイトル画面から「LOCAL」モードを選択できる
- 交互にターンが回る
- スコアがリアルタイムで表示される
- 勝者が明確に表示される
- ネクストプレイを選択できる
- 同じ盤面で再挑戦できる

#### US-003: プレイヤーがCPUと対戦する
> **As a** プレイヤー  
> **I want to** CPUと対戦する  
> **So that** 相手がいない時でもプレイできる

**受入基準:**
- 難易度を選択できる（EASY / NORMAL / HARD）
- CPUが適切な思考時間で手を打つ
- CPUの強さが難易度に応じて変化する

#### US-004: プレイヤーが練習する
> **As a** プレイヤー  
> **I want to** ソロモードでプレイする  
> **So that** 詰めトライパスを練習できる

**受入基準:**
- ソロモードを選択できる
- 最終スコアが表示される
- 何度でもリセットして再挑戦できる
- 最初に赤コマを取ることはできない（リムービング時）
- 最後の移動で赤コマを取得した場合はスコアが0点になる

#### US-005: プレイヤーがゲームをリセットする
> **As a** プレイヤー  
> **I want to** ゲームをリセットする  
> **So that** 新しいゲームを始められる

**受入基準:**
- ゲーム中に「RESET」ボタンが表示される
- ゲームオーバー画面に「ネクストプレイ」ボタンが表示される
- ゲームオーバー画面に「同じ盤でプレイ」ボタンが表示される
- 「同じ盤でプレイ」を選択すると、初期配置を維持してリセットされる

#### US-006: プレイヤーがタイトルに戻る
> **As a** プレイヤー  
> **I want to** タイトル画面に戻る  
> **So that** 別のゲームモードを選択できる

**受入基準:**
- ゲーム中に「TITLE」ボタンが表示される
- ボタンクリックでタイトル画面に遷移する
- ゲームの進行状況はリセットされる

#### US-007: プレイヤーがサウンドを調整する
> **As a** プレイヤー
> **I want to** 音量やオンオフを切り替える
> **So that** 環境に合わせてプレイできる

**受入基準:**
- ゲーム開始時にサウンド設定モーダルを表示（次回以降非表示可）
- 画面右上の音量アイコンから「大・小・消」を切り替え可能
- BGMがループ再生される

#### US-008: チュートリアルで学ぶ
> **As a** 初心者
> **I want to** インタラクティブなチュートリアルをプレイする
> **So that** 実際に動かしながらルールを理解できる

**受入基準:**
- タイトル画面から「TUTORIAL」を選択できる
- ガイドメッセージに従って順を追って操作できる
- ステップバイステップでルールが説明される

## 12. 用語集

| 用語 | 定義 |
|-----|------|
| **トライパス（TRYPAS）** | 日本発祥のボードゲーム。TRY（挑戦）とPASS（通過）の造語 |
| **陣形** | ゲーム開始時のコマ配置パターン |
| **スポット** | ボード上のコマ配置可能な位置（15箇所） |
| **ジャンプ** | コマを飛び越える移動動作 |
| **キャプチャ** | 飛び越えたコマを取得すること |
| **赤コマボーナス** | 赤コマを取得すると追加ターンを得られる特殊ルール |
| **詰めトライパス** | 終盤数手前からの練習問題（ソロモードに相当） |
| **ローカル対戦** | 同一デバイス上での2人対戦 |
| **フェーズ** | ゲームの進行段階（REMOVING / PLAYING / GAME_OVER） |
| **リムービング（REMOVING）** | ゲーム開始時にコマを1つ削除するフェーズ |
| **プレイング（PLAYING）** | 通常のゲームプレイフェーズ |
| **手詰まり** | 有効な移動が存在しない状態（ゲーム終了条件） |
